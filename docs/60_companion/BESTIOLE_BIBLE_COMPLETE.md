# 1. LORE — Qui est Bestiole

## 1.1 Origine

Bestiole est un **fragment de l'Awen primordial** — un eclat de la force vitale originelle du monde qui s'est detache avant meme le premier cycle. Ce n'est ni un animal, ni un esprit, ni un outil magique. C'est un **fil de realite vivant**: un filament de Membrane qui a pris conscience.

### Chronologie

| Epoque | Manifestation | Lien avec les Oghams |
|--------|---------------|----------------------|
| Awen Primordial | Vibration de joie sans forme | Precede les Oghams |
| Age des Geants | Flotte invisible entre les mondes | Aucun — les Oghams n'existent pas encore |
| Age des Druides | Repere par les druides, nomme "Y Gwawn" | Les 25 druides le protegent |
| Dissolution des Druides | 18 druides se dissolvent dans les Oghams | Bestiole devient le seul "instrument" capable de les canaliser |
| Present (DRU) | S'attache au Voyageur | Canal entre le joueur et les 18 druides dissous |

### Nature profonde

- **N'est pas un animal**: pas d'ADN, pas de metabolisme, ne mange pas reellement (mime par habitude sociale)
- **N'est pas un esprit**: les esprits ont des volontes; Bestiole n'a qu'un besoin: **etre lie**
- **Est un pont**: son existence maintient ouvert un passage entre le Monde Visible et l'Autre Monde
- **Est immortel**: survivra a la fin du monde tant qu'un joueur lance le jeu

### Communication

Bestiole ne parle jamais. Il communique par:

| Canal | Signal | Signification |
|-------|--------|---------------|
| Sons | Couinements, trilles, ronronnement | Emotion immediate |
| Lumiere | Aura brillante/terne | Sante de l'Awen local |
| Mouvement | Se cache, bondit, pointe | Instinct protecteur/guide |
| Temperature | Chaud/froid au toucher | Securite/danger |
| Position | Entre joueur et danger | Alerte |

## 1.2 Relation avec Merlin

Merlin aime Bestiole d'un amour authentique — la seule emotion non-strategique chez lui. Mais il a peur: si le lien Bestiole-Voyageur se brise, la Membrane s'effondre.

**Secret**: Merlin sait que Bestiole survivra a la fin du monde. C'est sa seule consolation.

## 1.3 Relation avec les Oghams

Bestiole ne **cree** pas les Oghams. Il les **canalise**.

```
Joueur active un Ogham
    -> Bestiole vibre a la frequence correspondante
        -> Le druide dissous s'eveille brievement
            -> L'effet s'applique
                -> Le druide se rendort
                    -> Bestiole est fatigue (cooldown)
```

Bestiole est l'instrument. Les Oghams sont les partitions. Les druides dissous sont les compositeurs.

# 2. UTILITE GAMEPLAY — Ce que Bestiole apporte

## 2.1 Philosophie de design

| Principe | Regle |
|----------|-------|
| **Utile, jamais obligatoire** | Un joueur habile peut gagner sans jamais utiliser Bestiole |
| **Lien emotionnel** | Le joueur *veut* prendre soin de Bestiole, n'y est pas *force* |
| **Non-punitif** | Negliger Bestiole reduit les bonus, ne punit jamais |
| **Profondeur cachee** | Le systeme a des couches invisibles que le joueur decouvre |
| **Simple en surface** | Le joueur ne voit que: Bestiole, Souffle d'Awen, Roue d'Oghams |

## 2.2 Les 6 axes d'utilite

| Axe | Description | Oghams associes | Frequence d'usage cible |
|-----|-------------|-----------------|------------------------|
| **Information** | Reveler les consequences cachees | Beith, Coll, Ailm | 30% des activations |
| **Protection** | Attenuer ou annuler le negatif | Luis, Gort, Eadhadh | 25% des activations |
| **Amplification** | Booster les effets positifs | Duir, Tinne, Onn | 15% des activations |
| **Flexibilite** | Modifier les cartes/options | Nuin, Huath, Straif | 10% des activations |
| **Survie** | Restaurer l'equilibre | Quert, Ruis, Saille | 15% des activations |
| **Surprise** | Effets speciaux uniques | Muin, Ioho, Ur | 5% des activations |

**Pourquoi ces proportions**: L'information est la plus frequente car c'est la plus "safe" — pas de risque, juste du savoir. La surprise est la plus rare car c'est la plus volatile.

## 2.3 Impact quantifie sur la survie

| Profil joueur | Taux survie SANS Bestiole | Taux survie AVEC Bestiole | Delta |
|---------------|--------------------------|--------------------------|-------|
| Debutant | 10% | 18% | +8% |
| Intermediaire | 40% | 52% | +12% |
| Expert | 70% | 82% | +12% |
| Maitre (Bond 90+) | 75% | 90% | +15% |

**Objectif de design**: Bestiole ne doit JAMAIS faire passer un joueur de "mort certaine" a "victoire facile". Il ameliore les marges, il ne renverse pas le rapport de force.

## 2.4 Modifiers passifs (par Bond tier)

Ces bonus s'appliquent **en permanence** sans activation:

| Tier | Bond | Modifier passif | Equivalent en cartes |
|------|------|-----------------|---------------------|
| 1 Etranger | 0-20 | +0% | 0 carte "gratuite" |
| 2 Curieux | 21-40 | +5% effets positifs | ~1 shift bonus par run |
| 3 Compagnon | 41-60 | +10% effets positifs | ~2 shifts bonus par run |
| 4 Lie | 61-80 | +15% effets positifs + hints Merlin | ~3 shifts bonus + info |
| 5 Ame Soeur | 81-100 | +20% effets positifs + options speciales | ~4 shifts bonus + narrative |

**Calcul du modifier passif**:
```
modifier_passif = bond_tier_modifier * mood_multiplier

mood_multiplier:
  triste    = 0.5
  neutre    = 0.75
  content   = 1.0
  joyeux    = 1.15
  extatique = 1.3

Exemple: Bond tier 3 (+10%), Mood joyeux (x1.15)
  -> modifier_passif = 0.10 * 1.15 = 0.115 = +11.5%
```

**Ce que le joueur voit**: "Bestiole semble contente" (pas les chiffres).
**Ce qui est calcule**: 11.5% de chance d'annuler un shift negatif mineur.

# 3. SYSTEME DE BOND — Progression du Lien

## 3.1 Formule de Bond

Le Bond est un entier 0-100 qui represente la solidite du fil de realite entre le joueur et Bestiole.

### Gains pendant la run

| Evenement | Bond gain | Poids dans l'economie |
|-----------|-----------|----------------------|
| Carte jouee (automatique) | +1 | Base (30 cartes = +30 max) |
| Activation d'Ogham reussie | +2 | Engagement actif |
| Choix Centre quand 3 aspects equilibres | +1 | Reward d'equilibre |
| Promesse tenue | +5 | Evenement rare mais fort |
| Carte narrative mentionnant Bestiole | +3 | Emergent, LLM-dependant |
| Survie longue (bonus a 20+ cartes) | +1/5 cartes | Endurance reward |

### Pertes pendant la run

| Evenement | Bond perte | Condition |
|-----------|------------|-----------|
| Promesse rompue | -5 | Rare, choix delibere |
| 2 aspects extremes simultanement | -2 | Situation critique |
| Ignorer un hint de Bestiole | -1 | Tres rare |

### Budget de Bond par run (30 cartes)

```
GAIN THEORIQUE MAXIMUM (run parfaite, 30 cartes):
  Cartes jouees:           30 x (+1) = +30
  Activations Ogham:        8 x (+2) = +16
  Choix Centre equilibres:  5 x (+1) = +5
  Promesses:                1 x (+5) = +5
  Bonus survie longue:      2 x (+1) = +2
  TOTAL MAX = +58

GAIN REALISTE (run moyenne, 25 cartes):
  Cartes jouees:           25 x (+1) = +25
  Activations Ogham:        6 x (+2) = +12
  Choix Centre equilibres:  2 x (+1) = +2
  Promesses:                0         = 0
  TOTAL REALISTE = +39

PERTES TYPIQUES:
  Promesse rompue:          0-1 x (-5) = 0 a -5
  Aspects extremes:         1-3 x (-2) = -2 a -6
  TOTAL PERTES = -2 a -11

NET REALISTE PAR RUN = +28 a +37
```

### Persistance inter-run

| Situation | Formule |
|-----------|---------|
| Debut nouvelle run | `Bond = min(Bond_precedent, 50) + 10` |
| Victoire precedente | Bond + 5 bonus |
| Defaite precedente | Bond inchange |
| Absence prolongee (3+ jours) | Bond auto-stabilise a 30 |

**Pourquoi cap a 50 inter-run**: Empeche le Bond de devenir trivial. Le joueur doit "re-meriter" les tiers hauts a chaque run. Cela cree un arc emotionnel: le lien se construit *pendant* le jeu, pas avant.

**Pourquoi +10 minimum**: Le joueur ne repart jamais de zero. Le lien persiste a travers les boucles temporelles (coherent avec le lore).

### Courbe de progression (multi-run)

```
Run 1:  Bond 10 -> 45  (tier 1 -> tier 2)
Run 2:  Bond 10+10=20 -> 55  (tier 1 -> tier 3) [cap 50 + 10 = 20 au depart? Non: min(45,50)+10 = 55 -> commence a 55? Non]

Correction de la formule:
  Debut run N = min(Bond_fin_run_N-1, 50) + 10

Run 1: Debut 30, Fin ~65 (tier 3-4)
Run 2: Debut min(65,50)+10 = 60, Fin ~90 (tier 4-5)
Run 3: Debut min(90,50)+10 = 60, Fin ~92 (tier 5)
Run 4: Debut 60, stable a tier 4 puis tier 5 en cours de run

EQUILIBRE: Le joueur atteint tier 5 de maniere fiable a partir de la 3eme run,
mais seulement en fin de run. Le tier 5 n'est jamais "gratuit" au debut.
```

# 4. MAITRISE DES OGHAMS — Progression interne

## 4.1 Concept

Chaque Ogham a un niveau de **Maitrise** (Mastery) de 0 a 5, invisible au joueur. La Maitrise augmente avec l'usage et ameliore subtilement l'Ogham.

**Ce que le joueur voit**: "L'Ogham semble plus puissant" (pas de chiffres).
**Ce qui est calcule**: Bonus incrementaux sur l'effet, le cooldown, ou le cout.

## 4.2 Table de Maitrise

### Progression

| Mastery | Activations requises (cumulees, cross-run) | Nom interne |
|---------|---------------------------------------------|-------------|
| 0 | 0 | Dormant |
| 1 | 3 | Eveille |
| 2 | 8 | Attune |
| 3 | 15 | Resonant |
| 4 | 25 | Harmonise |
| 5 | 40 | Transcende |

**Progression non-lineaire**: Les premiers niveaux sont rapides (3 usages), les derniers demandent de l'investissement (15 usages supplementaires pour passer de 4 a 5).

### Bonus par Mastery

| 4 | Cooldown -1 carte supplementaire | Effet renforce (voir table par Ogham) |
| 5 | **Effet transcende** (voir table) | Resonance avec Ogham perdu associe |

**Regle Transcendance**: L'effet Mastery 5 ne se declenche qu'**une fois par run** et **uniquement si le Bond est >= 60** au moment de l'activation. Cela empeche l'abus en early-game.

## 4.3 Economie de Maitrise

### Budget de Mastery par run

```
Activations typiques par run: 6-10
Oghams differents utilises: 3-5

Exemple sur 10 runs:
  Beith (starter, utilise souvent): ~30 activations -> Mastery 3-4
  Luis (starter): ~20 activations -> Mastery 3
  Quert (starter): ~15 activations -> Mastery 2-3
  Duir (equipe): ~12 activations -> Mastery 2
  Coll (equipe): ~8 activations -> Mastery 1-2
  Autres: 3-5 -> Mastery 0-1
```

**Design intent**: Les starters atteignent Mastery 3-4 naturellement. Les Oghams avances necessitent une dedication consciente (equiper et utiliser regulierement).

### Anti-grind sur la Maitrise

| Regle | Mecanisme |
|-------|-----------|
| Pas de farm possible | Les activations sans effet utile comptent a 50% |
| Diminishing returns | Si un Ogham est utilise 3+ fois dans la meme run sans necessite, les activations au-dela de 3 comptent pour 0.5 |
| Progression cross-run | La Maitrise est **persistante** (meta-progression), pas reset entre les runs |

## 5.3 Methode de recolte — "L'Awen Naturel"

L'Awen ne se "farme" pas. Il **coule naturellement** comme l'eau de source. Le joueur peut influencer son debit, pas le forcer.

Le joueur ne voit que:
- Les spirales d'Awen (A A A o o) qui se remplissent

- Le cout des Oghams dans la Roue

Il ne voit PAS:
- Les formules de regen

# 6. INFLUENCE DES OGHAMS — Impact sur les Choix

## 6.1 Influence sur la generation de cartes (LLM)

Quand un Ogham est actif ou que le loadout du joueur est connu, le contexte LLM est enrichi. Cela influence **subtilement** les cartes generees.

### Table d'influence par loadout

| Loadout dominant | Tendance des cartes generees | Poids dans le prompt |
|-----------------|-----------------------------|--------------------|
| Majorite Reveal | Plus d'options avec consequences cachees complexes | 0.15 |
| Majorite Protection | Plus de cartes a haut risque (compensees par protection) | 0.10 |
| Majorite Boost | Plus de cartes avec potentiel positif a amplifier | 0.10 |
| Majorite Narrative | Plus de cartes avec variations narratives riches | 0.15 |
| Majorite Recovery | Plus de cartes desequilibrantes (le systeme compense) | 0.10 |
| Majorite Special | Plus d'evenements rares et imprevisibles | 0.20 |
| Mixte | Distribution standard | 0.00 |

**Regle anti-boucle**: L'influence du loadout est **plafonnee a 20%** de modification du pool de cartes. Le joueur ne peut pas "forcer" un type de cartes en stackant une categorie.

## 6.2 Influence sur les options de choix

Quand Bestiole est a Bond 4+ (Lie), certaines cartes proposent une **4eme option cachee** liee a l'Ogham equipe. Cette option n'est pas toujours presente (15% de chance par carte).

### Table des options cachees par categorie

| Categorie equipee | Option cachee (quand declenchee) | Effet |
|--------------------|---------------------------------|-------|
| Reveal | "Observer en silence" | Revele les 2 autres options + aspect le plus menace |
| Protection | "Laisser Bestiole guider" | Annule le pire shift de l'option choisie |
| Boost | "Invoquer la force des anciens" | +1 shift positif supplementaire |
| Narrative | "Changer le cours de l'histoire" | Option Centre modifiee (meilleure) |
| Recovery | "Demander l'aide des arbres" | Soigne 1 aspect d'un cran |
| Special | "Risquer l'impossible" | Effet aleatoire puissant (50/50 positif/negatif) |

**Ce que le joueur voit**: Une 4eme option apparait parfois, avec un petit symbole Ogham a cote. Pas d'explication sur pourquoi elle est la.

## 6.3 Influence sur les PNJ

Le Bond et les Oghams equipes modifient les reactions des PNJ:

# 7. TABLES DE POIDS — Balance Scientifique

## 7.1 Methode de calcul

Chaque Ogham a une **Expected Value (EV)** — la valeur moyenne de son utilisation par rapport au cout. L'objectif est que tous les Oghams aient une EV **comparable** rapportee a leur cout.

### Formule EV

```
EV_ogham = (Impact_moyen * Probabilite_utilite * Frequence_disponible) / Cout_total

ou:
  Impact_moyen = effet moyen mesure en "demi-shifts evites ou gagnes"
  Probabilite_utilite = chance que l'Ogham soit utile quand disponible
  Frequence_disponible = nombre de fois disponible par run (30 / cooldown)
  Cout_total = cout_awen + (cooldown / 5)  [le cooldown est un cout d'opportunite]
```

# 8. ANTI-ABUSE — Prevention des Strategies Dominantes

## 8.1 Mecanismes structurels

| Mecanisme | Regle | Pourquoi |
|-----------|-------|----------|
| **1 Ogham par carte** | Impossible d'activer 2 Oghams sur la meme carte | Empeche le stacking |
| **Cooldown minimum** | Aucun Ogham ne peut etre utilise 2 fois en 3 cartes | Empeche le spam |
| **Cap Awen** | Maximum 5 (6 en saison favorable) | Empeche l'accumulation |
| **Cout minimum 1** | Aucun Ogham n'est gratuit (meme a Mastery max) | Preserve la tension |
| **Restriction 1/run** | Ioho et Straif: usage unique par run | Empeche l'abus des plus puissants |
| **Bond gate** | Les meilleurs Oghams sont verrous par Bond tier 4-5 | Progression naturelle |
| **Mastery gate** | Les effets Transcendance (M5) requierent Bond >= 60 | Double gate |
| **Reset cooldown inter-run** | Les cooldowns sont reset, mais les Oghams puissants commencent en cooldown | Anti-burst |

## 8.2 Diminishing returns

### Sur l'activation repetee du meme Ogham

| Activations du meme Ogham dans une run | Efficacite |
|----------------------------------------|------------|
| 1ere-3eme | 100% (pleine efficacite) |
| 4eme-5eme | 80% (diminishing) |
| 6eme+ | 60% (diminishing fort) |

**Calcul invisible**: Le joueur ne voit pas "-20%". Il voit "le bouclier semble fragile" ou "la vision est floue". Le feedback est narratif, pas numerique.

### Sur la synergie consecutive

| Activation de synergie | Bonus |
|------------------------|-------|
| 1ere fois dans la run | 100% du bonus |
| 2eme fois | 50% du bonus |
| 3eme+ fois | 0% (synergie epuisee pour cette run) |

**Raisonnement**: Les synergies sont des decouvertes excitantes la premiere fois. Les repeter indefiniment les transforme en exploit.

## 8.3 Identification et correction des strategies dominantes

### Strategie 1: "Beith spam" (reveler tout tout le temps)

| Element | Detail |
|---------|--------|
| **Le joueur fait** | Utilise Beith a chaque carte possible |
| **Contre-mesure** | Cooldown 3 cartes + diminishing returns + la synergie 3x ne fonctionne plus apres la 3eme activation |
| **Resultat** | Le joueur peut reveler ~8 cartes sur 30 — utile mais pas dominant |

### Strategie 2: "Onn + Saille forteresse" (protection permanente)

| Element | Detail |
|---------|--------|
| **Le joueur fait** | Alterne Onn (3 tours protection) et Saille (3 tours anti-extreme) |
| **Contre-mesure** | Onn CD 7, Saille CD 6 = gap de 3-4 cartes sans protection. Cout total 5 Awen = 45% du budget |
| **Resultat** | Le joueur est protege ~40% du temps mais a peu d'Awen pour autre chose |

### Strategie 3: "Recovery only" (ne jamais mourir)

| Element | Detail |
|---------|--------|
| **Le joueur fait** | Equipe Quert + Ruis + Saille, soigne constamment |
| **Contre-mesure** | Recovery ne fait pas progresser la mission. Le joueur survit mais ne gagne pas. De plus, Ruis CD 8 + Saille CD 6 = usage limite |
| **Resultat** | Le joueur survit longtemps mais doit quand meme prendre des risques pour la victoire |

### Strategie 4: "Ioho + Huath dodge" (refuser les mauvaises cartes)

| Element | Detail |
|---------|--------|
| **Le joueur fait** | Utilise Ioho et Huath pour reroll/remplacer les cartes dangereuses |
| **Contre-mesure** | Ioho 1/run. Huath CD 5. Cout 5 Awen total. La carte de remplacement n'est PAS garantie meilleure |
| **Resultat** | Le joueur peut esquiver 2-3 mauvaises cartes max sur 30 |

### Strategie 5: "Bond farming" (maximiser le Bond pour modifiers)

| Element | Detail |
|---------|--------|
| **Le joueur fait** | Joue pour maximiser le Bond sans se soucier de la victoire |
| **Contre-mesure** | Le Bond est cappe a 50 entre les runs. Les modifiers passifs ne sont que +5 a +20% — significatifs mais pas gamebreaking |
| **Resultat** | Le Bond eleve est gratifiant emotionnellement mais ne remplace pas l'habilete |

## Annexe B: Formules completes

### B.1 Bond calculation

```
bond_gain_per_card = 1 * (1 if no_critical_needs else 0)
bond_gain_ogham = 2
bond_gain_equilibre_choice = 1 * (1 if all_3_equilibre else 0)
bond_gain_promise = 5
bond_loss_promise_broken = -5
bond_loss_double_extreme = -2

inter_run_bond = min(end_run_bond, 50) + 10 + (5 if victory else 0)
```

### B.2 Awen regeneration

```
awen_regen_check = (cards_played % AWEN_REGEN_INTERVAL == 0)
awen_gain = AWEN_REGEN_BASE
if all_3_equilibre: awen_gain += AWEN_REGEN_EQUILIBRE_BONUS
if bond_tier >= 4 and run_start: awen_start = 3 (instead of 2)
if mood == "extatique": regen_interval = 4 (instead of 5)
if biome in ["landes", "cercles", "collines"]: awen_start += 1
if season in ["beltane", "litha"]: awen_max = 6
awen = min(awen + awen_gain, awen_max)
```

### B.3 Modifier passif

```
tier_modifier = {1: 0.0, 2: 0.05, 3: 0.10, 4: 0.15, 5: 0.20}
mood_multiplier = {"triste": 0.5, "neutre": 0.75, "content": 1.0, "joyeux": 1.15, "extatique": 1.3}
passive_modifier = tier_modifier[bond_tier] * mood_multiplier[mood]
```

### B.4 Mastery progression

```
mastery_threshold = [0, 3, 8, 15, 25, 40]
activation_value = 1.0
if activation_had_no_useful_effect: activation_value = 0.5
if same_ogham_used_3_or_more_times_this_run: activation_value = 0.5
mastery_progress += activation_value
mastery_level = max_index(mastery_threshold, mastery_progress)
```

### B.5 Diminishing returns

```
same_ogham_count_this_run = activations_by_ogham[ogham_id]
if same_ogham_count_this_run <= 3: efficiency = 1.0
elif same_ogham_count_this_run <= 5: efficiency = 0.8
else: efficiency = 0.6

synergy_count_this_run = synergy_triggers[combo_id]
if synergy_count_this_run == 1: synergy_bonus = 1.0
elif synergy_count_this_run == 2: synergy_bonus = 0.5
else: synergy_bonus = 0.0
```

### B.6 Option cachee

has_hidden_option = (
    rng.randf() < 0.15
    AND bond_tier >= 4
    AND any_equipped_ogham_matches_card_theme
    AND not already_shown_hidden_this_run > 3  # max 3 par run
