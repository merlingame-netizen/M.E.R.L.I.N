# World Rules (Bretagne Mystique)

Purpose
- Game world rules and content data in JSON.
- Content only (no code).

Files
- world_rules.json: global rules (time model, progression pace, promises, conflict, mystery)
- hourly_facts.json: 24 hourly modifiers, hours never block spawns (bonuses only)
- calendar_2026.json: dated events for 2026 with season locks and hour bonuses
- achievements.json: Gloire points, tiers, unlocks, and achievements
- GAMEPLAY_LOOP_ROGUELITE.md: core roguelite loop, quests, and discovery structure
- HOURLY_SLICES_GUIDE.md: 24h slices table, hour transition rules, JSON structure
- MERLIN_NARRATIVE_TEMPLATES.md: event type templates for unique narration
- MERLIN_MEMORY_RULES.md: what Merlin remembers during a run
- MERLIN_EVENT_EXAMPLES.md: example events with varied narration
- World_Rules_Overview.md: consolidated human readable rules document
- CHANGELOG.md: change log for rule and content updates
- DECISION_LOG.md: decisions and rationale for world rules
- TUNING_SHEET.md: balancing quick reference
- TEST_MATRIX.md: QA and validation checklist
- KNOWN_RISKS.md: risks and mitigations
- CALENDAR_2026_PRINT.md: printable calendar summary
- EVENT_TAG_GLOSSARY.md: tag meanings for events and spawns
- PROMISE_PLAYBOOK.md: promise examples and handling
- UTILITY_UNLOCK_MAP.md: achievements to unlock mapping

Notes
- Promises are heavy: failure creates Ogham debt and conflict pressure.
- Merlin sees all; lying to Merlin is nearly impossible.
- Progression is fast per day but full completion takes many seasons.
- Rewards are utilitarian tools and perks.
