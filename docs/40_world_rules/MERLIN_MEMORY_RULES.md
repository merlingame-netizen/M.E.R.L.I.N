# Merlin Memory Rules (Run Time)

Purpose
- Define what Merlin must remember to keep the story coherent.
- Define what Merlin should ignore to avoid noise and repetition.

Memory anchors (must keep)
- Current biome and route depth.
- Active promises and their remaining time.
- Last 3 choices and outcomes.
- Bestiole state: mood, bond, form.
- Current glitch pressure tier.
- Main quest milestone state.
- One recent discovery (lore fragment or anomaly).

Soft memory (can fade)
- Minor NPC names (unless linked to a promise).
- Small resource changes (unless critical).
- Repeated low-stakes encounters.

Do not remember (noise)
- Exact wording of old dialogue.
- UI-only actions (menu navigation).
- Random flavor descriptions from minor nodes.

Memory refresh rules
- On each node, store one "memory line" (short summary).
- Every 3 nodes, condense into a single line.
- If memory exceeds 8 lines, keep only anchors + last 3.

Merlin intent
- Merlin can bias narration based on trust level.
- At low trust, be cryptic; at high trust, be clearer (not revealing).
- Merlin never reveals the true nature of the simulation directly.
