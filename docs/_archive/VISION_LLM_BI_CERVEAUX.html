B5. Limites Realistes du 1.5B
Honnetete technique Qwen 2.5-1.5B est un nano-modele. Il excelle dans un cadre contraint mais a des limites fondamentales qu'aucun prompt engineering ne peut resoudre. Le code et le LoRA compensent.

Ce qu'il fait BIEN

CapaciteConditionQualite
Texte court en francais (50-150 tok)Prompt system < 150 tokens90%+
Format structure (A/B/C options)1 seul exemple dans le prompt85%+
JSON simple (GBNF contraint)Grammaire GBNF activee95%+
Vocabulaire injecte (biome, Ogham)Mots-cles dans le prompt user80%+
Variation narrative (Jaccard < 0.7)repetition_penalty ≥ 1.35OK
Reponse rapide (< 8s warm)max_tokens ≤ 150, Ollama17.8 tok/s

Ce qu'il fait MOYENNEMENT → Ameliorable par LoRA

CapaciteProblemeSolution LoRA
Vocabulaire celtique/druidiqueConnait les mots mais ne les utilise pas spontanementFine-tune sur corpus mythologique (500+ textes)
Persona Merlin coherentDerive vers un ton generique apres 3-4 cartesFine-tune sur dialogues Merlin rediges (300+ echanges)
Dilemmes moraux nuancesOptions trop symetriques (bon/neutre/mauvais)Fine-tune sur dilemmes avec couts reels sur chaque option
Descriptions sensoriellesTexte generique ("une foret sombre")Fine-tune sur descriptions biome (50+ textes par biome)
Adaptation au profil joueurIgnore le contexte profil dans le promptFine-tune avec exemples prompt+profil → texte adapte

Ce qu'il NE PEUT PAS faire → Compense par CODE

LimitationRaison fondamentaleCompensation CODE
Planifier sur plusieurs cartesPas de memoire inter-appels, context 4096 tokArc State Machine (CODE gere SETUP → CLIMAX → RESOLUTION)
Se souvenir des choix passesChaque appel est statelessRAG v2.0 injecte les choix pertinents dans le prompt
Auto-evaluer sa qualitePas de capacite meta-cognitive a 1.5BQuality Judge (CODE) verifie post-generation
Equilibrer les aspects TriadeNe comprend pas les regles de jeuGM + DifficultyAdapter (CODE) calcule les effets
Detecter la frustration joueurPas d'acces aux metriques de sessionSessionRegistry (CODE) detecte et injecte dans prompt
Gerer la tension narrativePas de notion de progression dramatiqueNarrativeRegistry.global_tension (CODE) controle l'escalade
Philosophie : LLM = Plume, CODE = Cerveau Le 1.5B est un excellent generateur de texte contraint. Toute logique de jeu, equilibrage, memoire et planification est geree en GDScript deterministe. Le LLM n'est jamais responsable de decisions strategiques.

D2. Prompt Engineering Avance
Optimisation des prompts pour Qwen 2.5-1.5B Chaque token compte. Les prompts ci-dessous sont calibres pour maximiser la qualite de sortie dans le budget token du nano-modele.

Prompt Narrator Optimise

ComposantBudgetContenu
System prompt~150 tokensRole Merlin, ton druidique, format A/B/C, regles de longueur, 1 exemple JSON
User prompt~100 tokensEtat Triade, biome, jour, profil joueur (1 ligne), effets GM (injectes), arc position
Params-T=0.60-0.70, top_p=0.90, max=180, rep_penalty=1.45, top_k=40

Template System Narrator:

Tu es Merlin, druide ancien. Raconte une scene en francais (80-150 mots). Ton: sage, poetique, avec humour subtil. Vocabulaire celtique bienvenu. Le voyageur fait face a un choix. Propose 3 options : A) B) C). Option B coute 1 souffle. Les effets mecaniques sont: {gm_effects} Biome: {biome}. Ambiance: {visual_tags.atmosphere}.

Prompt Game Master Optimise

ComposantBudgetContenu
System prompt~80 tokensRole GM, format JSON strict, effets SHIFT_ASPECT, visual_tags, audio_tags, 1 exemple
User prompt~60 tokensEtat Triade (3 aspects + valeurs), souffle, carte N, danger_level, arc_position
Params-T=0.15, top_p=0.8, max=80, rep_penalty=1.0, top_k=15, GBNF obligatoire

Template System GM:

Game Master. Genere JSON effets pour 3 options. Format: {"effects":[{"aspect":"Corps|Ame|Monde","dir":"up|down"},...], "visual":{"atmosphere":"...","light":"...","particles":"...","color_tint":"..."}, "audio":{"mood":"...","intensity":0.0-1.0,"elements":[...]}}

E. Roadmap par Phases

Implementation progressive en 5 phases, chacune livrant une version jouable amelioree.

Phase 0 - Stabilisation (2-3 semaines)
Fix bugs critiques A4 (MenuPrincipal crash, Store null).
Ameliorer feedback timeout Ollama (overlay "Merlin medite...").
Traduire labels generiques fallback en francais.
Valider 100% des scenes avec validate.bat Step 0-6.
Phase 1 - Game Master Intelligent (3-4 semaines)
Courbe de difficulte dynamique basee sur position Triade + carte count.
Detection de fin imminente: dilemme cornellien quand 2 aspects proches de l'extreme.
Anti-frustration: garantir 1 option stabilisante sur 3.
Tension narrative progressive: effets leger/moyen/intense selon progression.
Phase 2 - Narrator LoRA (4-6 semaines)
Constitution des datasets (corpus celtique + proverbes + sessions).
Fine-tuning LoRA Qwen 2.5-1.5B sur ton Merlin.
Benchmark GO/NO-GO par LoRA Evaluator agent.
Integration adapter GGUF dans Ollama.
Memoire narrative structuree (journal persistant).
Phase 3 - Synergie Bi-Cerveaux (3-4 semaines)
Protocole de communication Narrator-GM structure.
Coherence effets-narration verifiee par post-processing.
Arcs narratifs emergents (mini-arcs 3-5 cartes).
Adaptation biome profonde (vocabulaire specifique par environnement).
Dialogues Bestiole contextuels.
Phase 4 - Polish et Extras (2-3 semaines)
Voix narrative qui evolue avec la progression.
Descriptions sensorielles enrichies.
Evenements saisonniers celtiques.
Personnalisation Bestiole avancee.
Optimisation performances et tests de charge.
